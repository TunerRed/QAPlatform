### 后期检验已确认的bug
- 清空cache后不登录一下直接进问题详情会报错

  *CommonController第174行缺少了对user_id是否为空值的检验*
  
- 回复者前要加教师/学生标识，只有教师后面有用户名，学生则没有

  *添加该项时在用户名前使用了三元运算符，但判断的两个字符串（“STUDENT”和“TEACHER”）的后者与用户名被合起来当做第二个字符串*
  
  *即少加了一个括号*
  
- 查询数据库时经常出错，空指针异常

  *随机生成的数据中不知为何有一些用户id过大，需要在数据库中删除*
